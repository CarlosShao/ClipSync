# 2026-06-25 工作日志

## 已完成任务

### 1. 修复PostgreSQL容器启动失败
- **问题**: Docker容器挂载`scripts/init-db.sql`失败（Windows重启后路径访问问题）
- **解决**: 删除并重新创建容器（`docker compose down` + `docker compose up -d`）
- **结果**: 所有容器正常启动（PostgreSQL/Redis/API）

### 2. 修复验证码查询bug（auth.js）
- **问题**: 邮箱验证和密码重置查询中还有`used = FALSE`检查，导致验证码验证失败
- **修复**: 
  - 移除邮箱验证查询中的`AND used = FALSE`（第284行）
  - 移除邮箱验证降级查询中的`AND used = FALSE`（第292行）
  - 移除密码重置查询中的`AND used = FALSE`（第505行）

### 3. 修复加密密钥长度错误（encryption.js）
- **问题**: `ERR_CRYPTO_INVALID_KEYLEN` - 加密密钥长度不是恰好32字节
- **根因**: 
  - 环境变量名不匹配（`ENCRYPTION_KEY` vs `ENCRYPTION_MASTER_KEY`）
  - 密钥没有正确填充/截断到32字节
- **修复**:
  - 修改encryption.js读取`ENCRYPTION_KEY`环境变量（回退到`ENCRYPTION_MASTER_KEY`）
  - 实现`padOrTruncateKey()`函数确保密钥恰好32字节
  - 修改`createCipheriv`和`createDecipheriv`调用直接使用Buffer类型的MASTER_KEY

### 4. 修复数据库schema缺失列
- **问题**: `users`表缺少`tos_accepted_at`, `privacy_accepted_at`等列
- **修复**: 执行ALTER TABLE添加10个缺失列
  - tos_accepted_at, privacy_accepted_at, marketing_consent, birth_date, age_verified
  - is_active, deactivated_at, deactivation_reason, consent_updated_at, analytics_consent, functional_consent
