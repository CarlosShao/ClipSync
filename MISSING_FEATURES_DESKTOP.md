## 一、内存占用评估

### 当前观测
- 截图显示 **WebView2: ClipSync 占 147.1 MB**，这属于 **WebView2 桌面应用的常规范围**，不是异常泄漏。

### 为什么不算"非常轻量"
- Tauri 虽然是 Rust 主进程，但 UI 是完整的 WebView2（Edge 内核），本质上跑了一个 Vue 3 SPA。
- 内存包括：WebView2 渲染进程、V8 引擎、Vue 响应式数据、图片/Base64 缓存、剪贴板历史列表、IndexedDB/localStorage 等。
- 147MB 在 Electron/Tauri 类应用里算中等，但如果你期待的是"原生小工具几十 MB"，那确实偏高。

### 可优化点（已确认/待确认）
| 问题 | 来源 | 优先级 |
|------|------|--------|
| 图片预览 object URL 未 revoke，长会话内存泄漏 | pending-tasks-roadmap 4.1 | 🔴 P0 |
| 剪贴板列表无虚拟滚动，历史条目多会撑大 DOM | 观察项 | 🟡 P1 |
| 内容缓存 / offline-queue 跨用户未清理 | pending-tasks-roadmap 4.1 | 🟡 P1 |
| 图片缓存策略缺失（LRU/TTL 仅移动端有） | 架构债务 | 🟡 P1 |
| 无桌面端内存监控与告警 | commercial-readiness-review | 🟢 P2 |

## 总结：桌面端当前最优先补齐的 5 件事

1. **图片 object URL 释放 + 内存泄漏修复**（P0）
2. **Markdown 预览 + 代码语法高亮**（P0，roadmap 已标必须）
3. **端到端加密真正集成**（P1，但安全债务）
4. **模板功能 + 批量操作**（P0，高级剪贴板核心）
5. **AI/OCR 图片文字提取**（你明确提到的 AI 方向，P1）

如果按个人版上线标准，**功能上基本够用**；如果按成熟桌面剪贴板工具（Paste/CopyQ 级别）对标，**高级编辑、AI、模板、平台适配**都还有明显缺口。
