## 产品定位与 B2B 功能说明

### 产品定位
**ClipSync = 个人跨设备剪贴板同步工具**

- **核心场景**：个人用户在不同设备间同步剪贴板内容
- **目标用户**：程序员、设计师、办公人员等需要跨设备工作的个人
- **不是**：团队协作工具、企业知识库、代码片段管理平台

### 核心竞争力（与竞品的差异化）

ClipSync 的核心差异化应该是以下 4 点，所有开发工作围绕这 4 个方向展开：

| # | 核心优势 | 说明 | 竞品对比 |
|---|----------|------|----------|
| **1** | **同步速度（比 AirDrop 更方便）** | 实时推送，毫秒级延迟，无需手动操作 | AirDrop 需要手动选择文件/接收；ClipSync 自动同步 |
| **2** | **跨平台（Windows + macOS + Android + iOS）** | 全平台覆盖，不受操作系统限制 | Universal Clipboard 仅限 Apple 设备；Paste 仅 macOS |
| **3** | **隐私安全（端到端加密）** | ECDH 密钥交换，服务器无法查看明文内容 | 多数竞品无端到端加密；数据明文存储 |
| **4** | **离线支持（无网络时也能用）** | 本地缓存队列，网络恢复后自动同步 | 多数竞品离线即失效 |

> **开发原则**：每个新功能开发前，先回答——这个功能是否强化了上述 4 个核心优势？如果不是，降低优先级。

### 为什么 B2B 功能不是核心需求

**问题**：剪贴板工具需要 B2B 功能吗？

**答案**：❌ **不需要**。原因：

1. **剪贴板内容是个人隐私数据**
   - 用户复制的密码、信用卡号、私人消息不应该共享
   - 团队共享剪贴板有严重的安全和隐私风险

2. **B2B场景有更专业的工具**
   | B2B场景 | 专业工具 |
   |---------|----------|
   | 团队共享代码片段 | GitHub Gist, Notion, Slack |
   | 团队共享密码 | 1Password, Bitwarden 企业版 |
   | 团队共享文本/链接 | Slack, Microsoft Teams |
   | 团队协作 | Notion, Confluence, Figma |

3. **竞品证明**
   - 成功的剪贴板工具（Paste、Copied、Universal Clipboard）**全部是纯个人工具**
   - 没有一家做 B2B，也活得很滋润

4. **用户需求验证**
   - 如果你问用户："你需要团队共享剪贴板吗？"
   - 90% 的用户会说："不需要，这是我的私人数据"

**结论**：B2B 功能应该作为 **v2.0 扩展功能**（可选），而不是 launch  blockers。
