# 技术架构设计：跨设备剪贴板同步工具

## 1. 技术选型

### 1.1 前端框架

| 平台 | 推荐方案 | 备选方案 | 选择理由 |
|------|----------|----------|----------|
| **桌面端** | Tauri 2 | Electron | 体积小(7-15MB)、性能高、Rust后端 |
| **移动端** | Flutter | React Native | 跨平台UI一致、Material 3支持、性能优秀 |

**推荐组合**：Tauri 2 (桌面) + Flutter (移动端)

**优势**：
- Tauri：Rust后端，内存占用极低，适合常驻后台
- Flutter：一套代码iOS/Android，UI一致性好
- 两者都是2025-2026年最活跃的跨平台框架

### 1.2 同步方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **WebSocket云端中转** | 简单可靠、离线友好 | 需要服务器、隐私依赖服务商 | MVP阶段推荐 |
| **P2P点对点** | 隐私好、无需服务器 | 需设备同时在线、实现复杂 | V2版本 |
| **混合方案** | 兼顾两者优点 | 实现复杂度高 | 长期目标 |

**推荐方案**：MVP阶段使用WebSocket云端中转，V2版本增加P2P选项

### 1.3 AI方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **规则引擎** | 简单高效、无需模型 | 只能识别固定模式 | MVP阶段 |
| **本地轻量模型** | 隐私好、离线可用 | 模型大小、推理速度 | V2版本 |
| **云端API** | 能力强、更新快 | 隐私风险、网络依赖 | 可选增强 |

**推荐方案**：MVP阶段使用规则引擎，V2版本增加本地轻量模型

### 1.4 后端技术栈

| 组件 | 技术选择 | 说明 |
|------|----------|------|
| **API服务** | Node.js + Express/Fastify | 快速开发、生态完善 |
| **数据库** | PostgreSQL + Redis | 关系型+缓存 |
| **消息队列** | Redis Pub/Sub | 实时消息推送 |
| **文件存储** | 阿里云OSS/腾讯云COS | 图片文件存储 |

### 2.2 同步流程

```
设备A复制内容 → 客户端监听剪贴板变化
      ↓
内容加密(AES-256) → 上传到云端服务器
      ↓
服务器存储 → 推送到用户其他在线设备
      ↓
设备B接收 → 解密 → 写入本地剪贴板
```

### 2.3 数据安全

| 安全措施 | 说明 |
|----------|------|
| **传输加密** | TLS 1.3 + WebSocket加密 |
| **存储加密** | AES-256端到端加密 |
| **密钥管理** | 客户端生成密钥，服务器不存储明文 |
| **数据隔离** | 每用户独立加密密钥 |
| **自动过期** | 可配置历史记录保留时间 |

## 3. 数据库设计

### 3.1 用户表 (users)
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY,
    phone VARCHAR(20) UNIQUE,
    email VARCHAR(255) UNIQUE,
    password_hash VARCHAR(255),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### 3.2 设备表 (devices)
```sql
CREATE TABLE devices (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    device_name VARCHAR(100),
    device_type ENUM('windows', 'mac', 'ios', 'android'),
    platform_version VARCHAR(50),
    last_active_at TIMESTAMP,
    created_at TIMESTAMP
);
```

### 3.3 剪贴板记录表 (clipboard_items)
```sql
CREATE TABLE clipboard_items (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    content_type ENUM('text', 'image', 'file'),
    content_text TEXT,
    content_url VARCHAR(500),
    file_size INTEGER,
    is_favorite BOOLEAN DEFAULT false,
    tags JSONB,
    ai_category VARCHAR(50),
    created_at TIMESTAMP,
    expires_at TIMESTAMP
);
```

### 3.4 设备同步状态表 (device_sync_state)
```sql
CREATE TABLE device_sync_state (
    device_id UUID REFERENCES devices(id),
    last_sync_at TIMESTAMP,
    sync_cursor BIGINT,
    PRIMARY KEY (device_id)
);
```

## 4. API设计

### 4.1 用户认证
```
POST /api/auth/register      # 注册
POST /api/auth/login         # 登录
POST /api/auth/refresh       # 刷新token
```

### 4.2 设备管理
```
POST /api/devices            # 注册设备
GET  /api/devices            # 获取设备列表
DELETE /api/devices/:id      # 移除设备
```

### 4.3 剪贴板同步
```
POST /api/clipboard          # 上传剪贴板内容
GET  /api/clipboard          # 获取历史记录
DELETE /api/clipboard/:id    # 删除记录
PUT  /api/clipboard/:id/favorite  # 收藏/取消收藏
```

### 4.4 WebSocket接口
```
WS /ws/sync                  # 实时同步连接
```

## 5. 性能优化

### 5.1 客户端优化
- **增量同步**：只同步新增/变化的内容
- **智能压缩**：图片自动压缩（保持质量，减小体积）
- **本地缓存**：最近内容本地缓存，减少网络请求
- **后台运行**：最小化系统资源占用

### 5.2 服务端优化
- **Redis缓存**：热点数据缓存
- **数据库索引**：优化查询性能
- **CDN加速**：静态资源和文件分发
- **负载均衡**：多实例部署

### 5.3 网络优化
- **WebSocket长连接**：减少连接建立开销
- **断线重连**：自动恢复连接
- **心跳检测**：保持连接活跃
- **压缩传输**：gzip压缩

## 8. 技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 跨平台兼容性 | 不同平台剪贴板API差异 | 抽象层封装、充分测试 |
| 实时同步延迟 | 用户体验下降 | 优化网络、本地缓存 |
| 数据安全 | 用户隐私泄露 | 端到端加密、密钥管理 |
| 服务器成本 | 盈利压力 | 优化存储、分层定价 |
| 应用商店审核 | 上架延迟 | 提前准备、合规设计 |
